## State machine

```
selected в”Ђв”Ђв–є validating в”Ђв”Ђв–є uploading в”Ђв”Ђв–є processing в”Ђв”Ђв–є ready
                в”‚                в”‚              в”‚
                в–ј                в–ј              в–ј
              failed          failed         failed

retry (from failed) в”Ђв”Ђв–є selected (re-enters the flow)
```

| Status | Description |
|--------|-------------|
| `selected` | File added, queued for an upload slot |
| `validating` | Running client-side validation (size, type, duration) |
| `uploading` | Upload in progress, `progress` updates via adapter callback |
| `processing` | Upload complete, backend is processing (transcoding, etc.) |
| `ready` | Playback URL available |
| `failed` | Validation, upload, or processing failed. `error` contains the reason |

## FileState

```typescript
type FileState = {
  id: string;
  ref: FileRef;
  status: FileStatus;
  progress: number;            // 0вЂ“100 during upload
  thumbnailUri: string | null; // see Previews
  playbackUrl: string | null;  // available when ready; see Previews
  videoId: string | null;      // backend video ID
  error: string | null;        // error message when failed
  statusDetail: string | null; // sub-status from StatusChecker
};
```

## Immutability rules

- Files in `processing` or `ready` cannot be removed вЂ” they're committed to the server. `removeFile` on these statuses is a no-op.
- The file list is an append-only record of uploads in the current session.
- `retry` resets a `failed` file back to `selected`, re-entering the flow.

## statusDetail

Populated during `processing` when the `StatusChecker` provides sub-status updates (e.g. `"Transcoding 480p"`). Render it in your UI when `file.status === "processing" && file.statusDetail`.
